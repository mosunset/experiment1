# Lesson26_3
## 1. InsertionSort，ShellSort，OriginalShellSort に対して要素数や並び順による違いから InsertionSort の問題点について説明せよ．
※ 要素数については Lesson26 3 1.java の arraySizeは 10，100，1000，10000 に変更すること．また並び順については Lesson26 3 1.java の 13 行目，14 行目のコメントアウトを変更して確かめる．


### A.
要素がほぼ昇順に並んでいる場合は早いが極端に離れている値が出ると遅い。降順に並んでいる場合は、ほとんどバブルソートと一緒になる。

## 2. ShellSort，OriginalShellSort に対して，配列の並び順による結果の違いをもとに originalShellSort の問題点について説明せよ．
※ この時，ShellSort，OriginalShellSort に対し Lesson26 3 2 クラスの display(int[] array) メソッドを追加し，適宜途中結果を出力して比較すること．

### A.
ShellSortでは降順に並んでいてもランダムでも比較回数はほぼ変わらなかったが、originalShellsortは1.5倍ぐらいに増えた。<br>
検索する幅が、originalShellsortは半分づつになるため偶数列の比較が多くなり遅い。Shellsortは、1/3づつになるから比較場所がかぶる回数が減るため。
